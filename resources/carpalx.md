# Carpalx - a keyboard leyout optimizer

## [Evaluationg keyboard leyouts](https://mk.bcgsc.ca/carpalx/?keyboard_evaluation)

タイピングの労力モデルを使って配列を評価する。

幾つかの指標を定義てし、理想的な配列ではどうあるべきか書かれている。

頻度について。

- 行の頻度: row_hが高くrow_bが低いのが理想。
- 手の頻度: h_lとh_rが近いのが理想。
- 指の頻度: 弱い指から強い指に頻度が増えるのが理想。小指、薬指から、中指と人差し指。

連続について。

- 手の連続: rh_l(n)、rh_r(n)
- 行の連続: rr_t(n) 、rr_h(n)、rr_b(n)
- 指の連続: rf(n)

すべての連続は少ない方がよい。ホーム行の連続は多いほうがいい。

- 行の跳躍: rj(n)

行の跳躍は、飛ばした段の数の合計。

### 所感

頻度については概ね異論なし。

手の連続が続いたほうがいいというのは交互打鍵を推奨しているので納得できない。

## [Typing effort model](https://mk.bcgsc.ca/carpalx/?typing_effort)

レイアウトの探索結果は、訓練テキスト、レイアウトの評価方法、探索アルゴリズムの三つで決まる。

訓練テキストや探索アルゴリズムの良し悪しは、評価方法に比べるとある程度決まっている。

優れたレイアウトの特性について、弱い指の使用を制限する、ホームポジションを多く使用するなど、多くの共通点がある。

carpalxのタイピングモデルはパラメータ化されており、自分の好みに合わせて調整できる。

トライアドに基づいて評価する。
