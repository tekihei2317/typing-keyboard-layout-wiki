# ぶな配列

月系の前置シフトの計算配列。

2文字の連節の頻度と、2打鍵の所要時間をもとに、打鍵にかかる時間を最小化するように配置を決定している。アルゴリズムは山登り法を利用している。

## 配列図

ぶな配列v2.0

```
【シフト無し】
こにはても りっしのきれ
でか☆なた くん★いとら
だがますょ るう、。

【☆シフト】
づ＿ぢぷぅ ぬぱげじゃ「
ばさ☆ゅぜ ひをめーぶ」
ぴぞ＿ゆヴ ずむぎぐぽ

【★シフト】
べぼほそへ ＿ぺぇえや
わせけどあ びつ★おろ
ふざごねよ みちぉぃぁ
```

配置しているのは、

- 清音46文字
- 濁音半濁音26文字（がざだばぱ+ゔ）
- 小書き9文字（ぁぃぅぇぉゃゅょっ）
- 記号5文字（、。ー「」）を配置。

の計86文字。シフト面を2つ用意しつつ同手シフトも使えばいい感じに収まる。単打で31、32キー目を使うのは月配列の伝統だろうか。

## 配列の作成方法について

### [ちょっと、かな配列 ぶな配列とミズナラ配列の作り方　その1](http://keybor.blog96.fc2.com/blog-entry-59.html)

2連節の頻度のデータは小梅配列のもの、2打鍵の所要時間は幸花配列のものを使用している。

> 文字入力時間の求め方は、2文字連接において一つの2文字連接を入力する時の2文字目のみを打ちきる時間の合計とします。

ある文章を打つのにかかる時間は、

```
\sum ((2連節の出現回数) x (2連節を打つとき、1文字目を打ってから2文字目を打つのにかかる時間))
```

で計算できる。これを指標として使っていると考えられる。

### [ちょっと、かな配列 ぶな配列とミズナラ配列の作り方　その2](http://keybor.blog96.fc2.com/blog-entry-60.html)

山登り方を工夫したアルゴリズムを使っていることが書かれている。具体的なアルゴリズムの説明は3~6に続く。
